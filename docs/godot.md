# Godot 4 для ПДД-тренажёра

## Зачем Godot (если есть CARLA)

CARLA = видео-контент (записал → смонтировал → выложил).
Godot = **интерактив** (ученик сам решает ситуации в браузере).

Два инструмента для двух задач. Не конкурируют.

## Сильные стороны для ПДД

- **Web export (WebGL)** — ученик открывает в браузере, ничего не ставит
- **Лёгкий** — билд весит мегабайты, не гигабайты
- **GDScript** — Python-подобный, быстрый onboarding
- **Path3D / PathFollow3D** — машины/пешеходы по заданным маршрутам
- **Movie Maker mode** — запись в видео (встроено с Godot 4.0)
- **Open source** — MIT, никаких лицензий

## Формат для ПДД

### Интерактивный квиз
```
Экран: 3D-перекрёсток сверху
Вопрос: "Кто проедет первым?"
Кнопки: [Машина A] [Машина B] [Пешеход]
Ученик нажимает → анимация правильного ответа
```

### Симулятор от первого лица
```
Ученик "за рулём" → подъезжает к перекрёстку
Светофор, знаки, пешеходы
Надо правильно: остановиться / проехать / уступить
Обратная связь в реальном времени
```

## Готовые ресурсы

- **Kenney Car Kit** (free) — low-poly машины
- **Kenney City Kit** (free) — здания, дороги
- **Godot Vehicle Physics** — VehicleBody3D (встроенный)
- **OSM → SUMO → Godot** pipeline — импорт реальных карт
- **Octodemy driving simulation** — серия туториалов

## Когда выбрать Godot вместо CARLA

| Критерий | CARLA | Godot |
|---|---|---|
| Видео-контент | ✅ | Можно, но зачем |
| Веб-тренажёр | ❌ | ✅ |
| Фотореализм | ✅ UE5 | 🟡 Стилизация |
| Размер билда | Гигабайты | Мегабайты |
| Скорость разработки | Средняя | Быстрая |
| Трафик-симуляция | Из коробки | Писать руками |
| Кроссплатформа | Linux/Win | Всё + Web |

## Рекомендация

Начать с CARLA для видео-уроков.
Godot подключить позже для интерактивного тренажёра,
переиспользуя сценарии (Python → GDScript — похожий синтаксис).
